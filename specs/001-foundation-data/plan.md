## API Contract: `/api/data`

**GET** `/api/data`

| Parameter | Type | Values | Default | Required |
|-----------|------|--------|---------|----------|
| `dataset` | string | `911` \| `crime` \| `mapping` | `911` | Yes |
| `neighborhood` | string | any neighborhood name | (all) | No |
| `limit` | number | 1вЂ“10000 | 1000 | No |

**Response**: `200 OK` вЂ” JSON array of typed records matching the dataset type
**Error**: `400` for invalid `dataset` value; `200` with empty array if filters match nothing

---

## Data Loading Strategy

1. **Real data**: CSV/GeoJSON files downloaded from `opendata.montgomeryal.gov` and placed in `src/data/`
2. **Fallback**: If files absent, generate realistic sample data matching schema вЂ” document in `DECISIONS.md`
3. **Parsing**: Use `papaparse` for CSV with `header: true` and `dynamicTyping: true`
4. **Column aliasing**: Map known portal column variants to canonical field names in `montgomeryData.ts`
5. **GeoJSON**: Native `JSON.parse` вЂ” no external library needed

## Key Implementation Notes

- `montgomery-data.ts` runs **server-side only** (no `"use client"`) вЂ” reads `fs` from Node.js
- Use `path.join(process.cwd(), 'src/data', filename)` for reliable file paths
- `buildAIContext()` must truncate at 50,000 characters with a trailing `вЂ¦\n[truncated]` marker
- Sample data (if used): 200+ rows, realistic Montgomery neighborhoods (Cloverdale, Midtown, East Montgomery, etc.), realistic call types (DISTURBANCE, TRAFFIC STOP, BURGLARY, ASSAULT, etc.)
